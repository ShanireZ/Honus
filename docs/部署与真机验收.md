# Horus 部署与真机验收指南

> 面向**部署 / 监考运维**。成品在 `dist/`（gitignored·不入库）：`dist/server`（监考服务器）+ `dist/client`（采集 Agent）。
> 设计权威见 [architecture-v0.2.md](architecture-v0.2.md)、身份/RBAC/灰度 [m4-identity-oidc.md](m4-identity-oidc.md)、采集端硬化 [m5-agent-hardening.md](m5-agent-hardening.md)。

## 0. 架构与数据流（一句话）
- **监考服务器**（一台笔记本）：收采集数据 + 分析 + 落库 + Web 看板。单场 30~60 人×2h **单机绰绰有余**。
- **采集 Agent**（每台考试机一个·需管理员）：采 OS 信号 + 截图 → 局域网上报服务器。
- **数据全程局域网内**；唯一可选出网 = 视觉 LLM 识图（送派生图·原图永不出网）。判题走**外部洛谷**（不在本系统内）。
- 成品**自包含**（self-contained·免装 .NET 运行时），单文件 exe。

## 1. 监考服务器部署（`dist/server`）

### 1.1 前置
- 一台 Windows 10/11 x64 笔记本；局域网固定 IP（示例 `192.168.1.10`）。
- 无需安装 .NET（自包含）。视觉/CLIP 的密钥加密用 DPAPI（本机绑定），故**在部署机本机配置**。

### 1.2 配置 `server.config.json`（与 exe 同目录）
关键项（完整注释见模板 / [../server/server.config.sample.json](../server/server.config.sample.json)）：

| 项 | 说明 |
|---|---|
| `urls` | 绑定地址。局域网 `http://0.0.0.0:8080`；监考员远端 OIDC 登录另需 https（见 1.5） |
| `pskBase64` | **采集鉴权共享密钥**（authMode=psk/both）。32 字节 base64，Agent 用**同一把** |
| `authMode` | `psk`（最简·默认）\| `oidc`（cpplearn per-user 身份）\| `both`（灰度共存）。见 §4 |
| `adminToken` | **管理端令牌**（adminAuthMode=token）。看板登录用。生产务必配（否则同网学员机可改配置/拉证据图） |
| `adminAuthMode` | `token`（默认·静态令牌）\| `oidc`（cpplearn 长老登录·见 §4） |
| `keystrokeSecretBase64` | 击键旁路 KSK（当前判题走外部洛谷·**无落点·可留空**） |
| `riskThreshold` | 入可疑队列的风险阈值（默认 50） |
| `visionProvider`/`visionBaseUrl`/`visionModel`/`visionApiKey` | **视觉 LLM 识图**（可选·唯一出网）。填明文 key，**启动自动加密**为 `visionApiKeyEnc` 并清明文 |
| `embedProvider`/`embedOnnxModelPath` | **CLIP 按图搜图**（可选）。见 §6 |
| `retentionDays` | 留存天数（默认 30·到期关键数据转 archive） |

### 1.3 生成密钥（PowerShell）
```powershell
# 32 字节 base64 PSK（采集共享密钥；Agent 用同一值）
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }))
# 管理令牌（随便一段高熵字符串即可）
[Convert]::ToBase64String((1..24 | ForEach-Object { Get-Random -Maximum 256 }))
```
- **视觉/CLIP 的 API key**：直接把明文填进 `visionApiKey`，首次启动会自动加密为 `visionApiKeyEnc` 并清空明文（DPAPI·本机绑定）。也可 `Horus.Server.exe protect-secret <明文>` 预生成密文。

### 1.4 启动
```
Horus.Server.exe                 # 默认读同目录 server.config.json
Horus.Server.exe D:\path\server.config.json   # 或显式指定
```
- 启动日志确认：`db=… 采集鉴权=开 管理鉴权=开`。
- 浏览器开 `http://192.168.1.10:8080` → 看板；输入管理令牌（或走 OIDC 登录）。
- **非 loopback 绑定却缺 PSK/管理令牌会拒绝启动**（fail-closed）——生产必须配齐。

### 1.5 HTTPS（仅当监考员用 cpplearn OIDC 登录看板 / 远端工作站时）
- `urls` 加 `https://0.0.0.0:8443`；`httpsSanHosts` 填服务器 LAN IP/主机名。启动**自动生成自签证书**；监考机首用点过浏览器警告或预装证书。

### 1.6 部署前一键预检
- 看板顶栏「**预检**」按钮（或 `GET /api/preflight`）：检查采集/管理端鉴权配置、cpplearn 可达（OIDC 模式）、视觉 key、active 考试白名单覆盖、both→oidc 迁移覆盖。**全绿再开考**。

## 2. 采集 Agent 部署（`dist/client`·每台考试机）

### 2.1 前置
- 每台考试机 Windows 10/11 x64；**必须管理员权限**（ETW / UIAutomation / WMI / 抓屏）。
- 有默认浏览器（OIDC 模式登录用）。

### 2.2 配置 `agent.config.json`（与 exe 同目录）
| 项 | 说明 |
|---|---|
| `examId` | 本场考试 id（与服务器建的考试一致） |
| `seatId` / `agentId` / `machineId` | 座位号 / Agent 标识 / 机器标签（每机唯一） |
| `serverWsBase` / `serverHttpBase` | 服务器地址 `ws://192.168.1.10:8080` / `http://192.168.1.10:8080` |
| `authMode` | `psk`（填下方 `psk`·与服务器同一把）\| `oidc`（填 `oidcIssuer`+`oidcClientId`·经浏览器登录 cpplearn） |
| `psk` | psk 模式：服务器 `pskBase64` 的**同一值** |
| `oidcIssuer` / `oidcClientId` | oidc 模式：cpplearn issuer（如 `https://betaoi.cc`）+ `horus-client` |
| `whitelistHosts` / `whitelistProcs` | 允许的判题站 / 进程（IDE/编译器等）——也可由服务器下发覆盖 |
| `targetHeight` / `webpQuality` / `baselineMinSeconds` / `baselineMaxSeconds` | 截图分辨率/质量/随机基线区间 |

### 2.3 启动
- **右键「以管理员身份运行」** `Horus.Agent.exe`（默认读同目录 `agent.config.json`）。
- `psk` 模式：直接连服务器。`oidc` 模式：弹系统浏览器登录 cpplearn → 授权 → 自动回连。
- 看板对应座位变**在线**。

### 2.4 保活 / 抗规避（M5·推荐·见 §5 验收）
三层纵深，任一层的兜底都是「关掉即心跳断 → 看板告警 → 物理监考介入」：
- **层2 兄弟看门狗**（无需装服务）：`Horus.Agent.exe --watchdog`（拉起并监控采集进程·被杀秒级重启）。
- **层1 Windows 服务（最强·标准账户杀不掉 LocalSystem 服务）**：
  ```
  Horus.Agent.exe install-service      # 管理员运行:注册 LocalSystem 服务·自动启动·拉起采集进用户会话
  Horus.Agent.exe uninstall-service    # 卸载
  ```
- **层3 心跳告警**：已内建，采集被关/挂起 → 看板座位离线/健康告警。

## 3. 开考标准流程
1. **服务器**建考试 + 下发白名单（看板 / `POST /api/exams/{id}/config`）。
2. 各考试机起 Agent（或 M5 服务已自启）。
3. **考前预检**（看板「预检」按钮）→ 全绿。
4. 开考：看板实时座位热力 + 可疑队列。**系统只初筛，处分由人裁决**。
5. 考后：看板结束考试（`status=ended`）。到期自动归档。

---

## 4. M4 身份层（OIDC / RBAC·可选进阶）
- **采集面 OIDC**（psk→oidc）：per-user 身份取代共享 PSK，闭合跨身份栽赃/seq 抢占。灰度验收清单见 [m4-identity-oidc.md §11](m4-identity-oidc.md)。
- **管理端 RBAC**（token→oidc）：cpplearn **长老 = 监考员**（有看板管理权限），弟子 = 考生。监考员用 cpplearn 账号登录看板（需 §1.5 HTTPS + cpplearn 侧 `horus-dashboard` client）。
- **两条轴独立**，可各自灰度。切 oidc 前用预检 `migration` 项确认在线座位全部迁移。

---

## 5. M5 采集端硬化·真机验收清单
（在真机逐条验，纯检测取证·不阻断作弊，只让规避暴露）
1. **服务保活**：`install-service` → start → 采集在**用户会话**起、能截屏（看板有该座位截图）。
2. **杀采集**：任务管理器结束 `Horus.Agent.exe` → 看门狗**秒级重拉**（看板短暂离线后恢复）。
3. **杀看门狗**：结束看门狗进程 → 采集端**重拉一个 adopt 看门狗**（层2 互拉）。
4. **标准账户**：以学员标准账户尝试停服务 → **停不掉**（LocalSystem）。
5. **防挂起**：休眠/睡眠后恢复 → 看板出现 `suspected_suspend` 健康告警。
6. **防遮蔽**：用纯色窗口盖住主屏 → `screenshot_obscured` 入可疑队列。
7. **防降权限**：非管理员启动 Agent → `capability_degraded` 告警（看板座位健康标）。

## 6. CLIP 按图搜图（可选）
- 规模小 → C# 暴力余弦，无需 sqlite-vec。仅嵌**证据图**。
- **嵌入源**：★实测小米 MiMo-V2.5 **无** `/v1/embeddings` 端点 → 走**本地 ONNX CLIP**（零出网·CPU·无需 GPU）。
- 配置：
  - `embedProvider": "onnx"`；
  - `embedOnnxModelPath`：CLIP ViT-B/32 **图像编码器** ONNX（512 维）。取法：HuggingFace `Xenova/clip-vit-base-patch32` 的 `onnx/vision_model.onnx`（~150MB），放到 dataDir 或绝对路径。
- 启动日志见「**ONNX CLIP 嵌入器就绪**」即成；看板证据图灯箱「**按图搜图**」捞全场相似帧。
- 部署前可 `Horus.Server.exe probe-embed [model]` 探测某云端点是否有图像 embedding 模型。

## 7. 常见故障排查
| 现象 | 原因 | 处置 |
|---|---|---|
| 服务器拒绝启动「未配置采集面鉴权/AdminToken」 | 非 loopback 绑定缺 PSK/令牌（fail-closed） | 配 `pskBase64` + `adminToken`；仅联调可 `allowInsecure:true` |
| 座位一直离线 | Agent 未起/网络不通/身份不符 | 管理员起 Agent；核对 serverWsBase/防火墙；examId/seatId 与服务器一致 |
| Agent 报 `capability_degraded admin` | 没以管理员运行 | 右键以管理员身份运行 / 走 M5 服务 |
| 监考员看板 OIDC 登录失败 | cpplearn 不可达 / horus-dashboard client 未配 / 回调非 https | 预检 `issuer_reachable`；配 cpplearn env；服务器起 https |
| `kid 未命中 JWKS` | cpplearn RS256 密钥轮换 | 同步新 JWKS（内联或重拉）+ 重启 |
| 按图搜图无结果 | 未配嵌入器 / 模型缺失 / 尚无证据图 | 配 `embedProvider=onnx`+模型；等有证据图被嵌入 |
| 视觉分析不出结果 | key 未解出 / 端点不通 | 预检 `vision` 项；查 baseUrl/model/key |

> 安全铁律：真实配置（PSK / 令牌 / API key）在 `*.config.json`（**已 gitignore·切勿提交/外发**）；分发只发 `*.sample.json` 模板。
